# OctoAcme Project Management Docs

Welcome to the OctoAcme Project Management documentation. This README is the central entry point for understanding how OctoAcme plans, executes, and continuously improves its projects.

## Purpose

These documents capture OctoAcme's end-to-end project management processes — from kicking off a new initiative to shipping and reflecting on it. They are intended to help new team members get up to speed quickly, give existing members a shared reference, and ensure consistent practices across teams.

Key principles at a glance:
- **Customer-first** value delivery
- **Iterative releases** with clear success metrics
- **Defined roles** and ownership at every stage
- **Structured risk management** and escalation paths
- **Regular communication** and retrospective feedback loops

## How to Propose Changes

Found something outdated or want to add new process guidance? Here's how:

1. **Open an issue** using the [Add/Update Content to Process Docs](../.github/ISSUE_TEMPLATE/add-update-content-to-process-docs.yml) issue template — describe what needs to change and why.
2. **Submit a PR** with your proposed changes, keeping each PR small and focused on a single document or topic.
3. All changes go through **CI checks** and require **at least one approval** before merge.

If you are unsure whether a change is appropriate, open an issue first to discuss it with the team.
